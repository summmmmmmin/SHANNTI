## 💡 왜 만들었나

헬스장에서 혼자 운동할 때 자세가 올바른지 실시간으로 확인할 방법이 없다. PT 트레이너는 비용이 크고, 운동 후 녹화 영상을 돌려보는 방식은 교정 타이밍을 놓친다. **카메라 하나로 운동하는 동안 즉각 피드백**을 주는 앱이 필요하다고 판단했다.

기존 AI 포즈 분류 접근법은 매 프레임 딥러닝 추론 연산이 발생해 실시간 반응에 부적합하다. SHANNTI는 MediaPipe 랜드마크 좌표를 받아 **코사인 제2법칙 수학 연산만으로** 각도를 계산하기 때문에 추론 비용이 없고, 500ms 간격의 실시간 피드백이 가능하다. 이 접근법을 논문으로 정리해 한국정보통신학회논문지(JKIICE)에 게재했다.

## 🔬 핵심 구현

### 1. 각도 계산 알고리즘 — 코사인 제2법칙

딥러닝 모델 없이 **수학 연산만으로** 실시간 자세를 판정하는 것이 이 프로젝트의 핵심 기술적 선택이다.

포즈 분류 모델을 쓰면 매 프레임 추론 연산이 발생하지만, 코사인 제2법칙은 연산 비용이 O(1)으로 고정되어 500ms 폴링도 거뜬하다.

**계산 방식**

세 관절점(예: 엉덩이 → 무릎 → 발목)을 입력받아, 각 점 간의 유클리드 거리를 구한 뒤 코사인 제2법칙으로 꼭짓점(무릎)의 각도를 계산한다.

> θ = arccos( (a² + b² - c²) / 2ab ) × (180/π)

결과 각도를 운동별 정상 범위와 비교해 즉시 판정한다.

---

### 2. 좌우 랜드마크 자동 선택

MediaPipe는 좌측/우측 랜드마크를 각각 따로 제공한다. 사용자가 카메라를 향해 어느 방향으로 서 있든 **유효 범위 안에 있는 쪽**을 자동으로 선택한다.

| 처리 단계 | 내용 |
|----------|------|
| ① 유효성 사전 계산 | 33개 랜드마크 각각에 대해 화면 유효 범위(-100 ~ 1100) 내 여부를 배열에 저장 |
| ② 좌우 순차 검사 | 어깨 · 엉덩이 · 무릎 랜드마크가 모두 유효한 방향(좌/우) 탐색 |
| ③ 자동 선택 | 유효한 방향 기준으로 자세 판정 실행 |
| ④ 미감지 처리 | 양쪽 모두 유효하지 않으면 ⚫ NOT DETECTED 반환 |

사용자가 카메라 방향을 신경 쓸 필요 없다.

---

### 3. MediaPipe 좌표 변환

MediaPipe는 정규화된 좌표(0~1)를 반환하며, **X/Y 축이 화면 기준과 반전**되어 있다. 다음 두 단계로 보정한다.

| 처리 | 내용 |
|------|------|
| 축 교환 | MediaPipe Y → 화면 X (좌우) · MediaPipe X → 화면 Y (상하) |
| 스케일링 | 0~1 정규화 좌표 → 0~1000 범위로 확대 (각도 계산 수치 안정성 확보) |

## 📐 운동별 관절 분석 기준

> **설계 철학:** 핵심 관절(무릎·척추)은 좁은 허용 범위로 엄격하게, 보조 관절(팔)은 체형 다양성을 수용하도록 넓은 범위를 적용했다.

### 🏋️ 스쿼트 (Squat)

| 부위 | 관절점 | 정상 범위 | 중요도 |
|------|--------|-----------|--------|
| **무릎** | 엉덩이(23) → 무릎(25) → 발목(27) | **30° ~ 90°** | ⭐ 핵심 |
| **척추** | 어깨(11) → 엉덩이(23) → 무릎(25) | **40° ~ 90°** | ⭐ 핵심 |
| 팔 | 어깨(11) → 팔꿈치(13) → 손목(15) | 160° ~ 200° | 보조 |

![Image](https://github.com/user-attachments/assets/4ee538f7-6a90-4b42-b538-b104be9f12ec)

### 🧘 플랭크 (Plank)

| 부위 | 관절점 | 정상 범위 | 중요도 |
|------|--------|-----------|--------|
| **팔꿈치** | 어깨 → 팔꿈치 → 손목 | **60° ~ 120°** | ⭐ 핵심 |
| **척추** | 귀(7) → 어깨(11) → 엉덩이(23) | **150° ~ 210°** | ⭐ 핵심 |

### 🚶 런지 (Lunge)

| 부위 | 정상 범위 | 중요도 |
|------|-----------|--------|
| **앞다리 각도** | **40° ~ 100°** | ⭐ 핵심 |
| **척추** | **150° ~ 210°** | ⭐ 핵심 |

### ⚖️ 밸런스 (Balance)

한쪽 발 랜드마크(발목 → 발끝) 감지 기반 균형 유지 여부 판정.
10초 타이머 동안 유효 랜드마크 지속 감지 여부로 합격/불합격 결정.

## 🔥 트러블슈팅

### ① 체형 다양성 — 단일 기준으로 자세 판정 불가

**문제** 스쿼트 시 팔을 앞으로 뻗거나 팔짱을 끼는 등 체형·운동 습관이 다양해, 모든 관절에 동일한 범위를 적용하면 정상 자세가 오류 판정되거나 잘못된 자세가 통과하는 문제 발생.

**해결** 관절별 중요도에 따라 허용 범위를 차등 적용. 무릎과 척추는 운동 효과·부상 방지에 직결되므로 범위를 엄격하게 제한하고, 팔은 범위를 넓혀 체형 다양성을 수용.

**결과** 팔 자세가 달라도 무릎·척추 기준만 충족하면 정상 인정. 다양한 운동 스타일을 허용하면서도 핵심 관절 기준은 유지.

---

### ② 딥러닝 추론 비용 — 실시간 피드백 지연

**문제** 포즈 분류에 딥러닝 모델을 사용할 경우 매 프레임 추론 연산이 발생해 실시간 피드백에 부적합한 지연 발생.

**해결** MediaPipe GPU 파이프라인으로 33개 랜드마크 좌표만 추출하고, 이후 판정은 **코사인 제2법칙 수학 연산만** 사용. 별도 분류 모델 없이 순수 삼각함수 연산으로 처리.

**결과** 모델 로딩 지연 제거. 판정 주기 500ms · 1회 세션 5,000ms 고정 윈도우. 이 접근법을 논문으로 정리해 JKIICE에 게재.

---

### ③ 좌우 비대칭 — 어느 쪽 기준으로 판정할지 모호

**문제** MediaPipe는 좌우 랜드마크를 각각 제공. 사용자가 카메라 방향에 따라 어느 쪽 랜드마크를 기준으로 판정할지 불명확. 잘못된 쪽 선택 시 각도 계산 오류 발생.

**해결** 좌측·우측 양쪽을 순서대로 검사해, 어깨·엉덩이·무릎 랜드마크가 유효 범위 내에 있는 방향을 자동 선택. 각 랜드마크 유효성을 배열에 사전 계산해 탐색 비용 최소화.

**결과** 카메라 방향과 무관하게 유효한 쪽 신체 기준으로 자동 판정. 사용자가 방향을 신경 쓸 필요 없음.
