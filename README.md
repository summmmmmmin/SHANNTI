## Ⅰ. 서론 🧭

현대 사회에서 **건강·웰빙**에 대한 관심이 높아지면서, 많은 사람이 규칙적인 운동을 통해 건강을 유지하려 한다. 그러나 바쁜 일상 속에서 **시간·비용**을 들여 운동하기 어렵고, 전문가 없이 **올바른 자세**를 유지하는 것도 쉽지 않다.

운동에서 올바른 자세는 매우 중요하다. 잘못된 자세는 운동 효과를 떨어뜨릴 뿐 아니라 **부상 위험**을 높여 역효과를 낼 수 있다. 따라서 본 연구는 사용자가 **스스로 정확한 자세를 습득·유지**할 수 있도록 돕는 애플리케이션의 개발과 기술적 구현을 다룬다. 본 애플리케이션은 **스쿼트, 플랭크, 런지, 밸런스** 등 주요 운동 자세를 분석하여 올바른 자세로 운동할 수 있도록 지원하며, **스마트폰·태블릿**에서 동작하여 **장소·시간 제약**을 줄인다.

기존 딥러닝 기반 자세 분석은 주로 **정·오 판별** 또는 **자세 간 유사도**에 초점을 맞추는 경우가 많다면, 본 연구는 **관절 각도를 세밀히 분석**하여 기준 자세와의 **작은 차이**까지 포착하고 교정한다. 이를 통해 운동 과정에서 **실시간·즉각적인 피드백**을 제공하여 운동 수행의 **정확도와 효율성**을 높이고자 한다.

## Ⅱ. 관련 연구 📚

운동과 기술의 융합을 통한 **건강 관리 솔루션** 연구가 활발하다. 다양한 기기를 활용한 운동 시스템은 운동 습관 개선과 효과적인 건강 관리 방법으로 주목받는다.

| 주제 | 요약 | 비고 |
|:---|:---|:---:|
| 🦴 **MediaPipe 자세 분석** | 주요 관절 포인트 좌표·각도 계산으로 자세 분석 가능 | 운동 일지·루틴 추천 등 연구 존재 |
| 😐 **얼굴 인식** | 로그인 편의·보안 고려 | 시니어 등 입력 어려움 완화 |
| 🎮 **운동+게임(게임화)** | 동기부여·지속성 향상 | e-health 재활 등에서 보고 |
| 🖱️ **아두이노 에어마우스** | 가속도·자이로로 저비용 입력 장치 | 조작 난이도·기준점 변화 이슈 가능 |

기존 연구들은 개인 맞춤형 운동 솔루션의 가능성을 보여주나, 본 논문은 이를 바탕으로 **보다 포괄적이고 사용자 친화적인** 운동 애플리케이션을 제시한다.

## Ⅲ. 시스템 설계 및 구현 🛠️

### 3.1 시스템 구조

본 시스템의 핵심은 다음과 같다.

| 구성 요소 | 역할 | 이모지 |
|:---|:---|:---:|
| **ML Kit** | 얼굴 인식 로그인 | 👤 |
| **MediaPipe Pose Detection** | 관절 각도 기반 자세 분석 | 🦴 |
| **아두이노 에어마우스** | 유산소 게임 입력(블루투스) | 🖱️ |
| **Firebase** | 사용자·운동 기록 저장 | ☁️ |

- 사용자 정보와 얼굴 정보는 **MediaPipe Face Detection**을 활용해 **Firebase**에 저장·관리한다.  
- 로그인 후 **MediaPipe Pose Detection**으로 자세 분석·운동을 수행한다.  
- 게임 실행 시 **가속도·자이로 센서**가 장착된 아두이노 에어마우스를 **블루투스**로 연결한다.  
- 운동·게임 종료 후 **운동 시간·소모 칼로리** 등이 Firebase에 저장되어 기록에서 확인한다.

### 3.2 영상 분석 기반 운동 자세 분석

#### 3.2.1 얼굴 인식 로그인 👤

얼굴 인식 로그인은 **생체 인식**을 이용한 인증 방식으로, 얼굴의 고유 특징을 분석해 신원을 확인한다. 본 애플리케이션은 **ML Kit**를 사용한다.

**처리 흐름 (요약)**

| 단계 | 내용 |
|:---:|:---|
| 1️⃣ | 카메라로 실시간 이미지 캡처 |
| 2️⃣ | 전처리(형식 변환, 크기·해상도 조정) |
| 3️⃣ | 학습된 모델로 얼굴 감지 |
| 4️⃣ | 바운딩 박스 생성 |
| 5️⃣ | 다수 얼굴 감지 가능하나 **단일 얼굴만** 감지하도록 수정 |
| 6️⃣ | 바운딩 박스 좌표 **[0,1] 정규화** + 랜드마크 식별 → 임베딩 벡터 |

> 📌 **Fig. 2** MLKit Face Detection (논문 원본 그림 참조)

**등록·로그인 절차**

| 구분 | 설명 |
|:---|:---|
| 📝 **등록** | 최초 캡처 → 전처리·감지·랜드마크 → 특징 **암호화 저장** |
| 🔓 **로그인** | 동일 전처리·감지 → DB 특징과 **유사도 비교** |

유사도는 **유클리드 거리** 기반으로 비교하며, 데이터셋 분석으로 **평균·표준편차**를 고려한 임계값을 실험적으로 적용한다. 논문에는 식 (1)으로 표현된다.

**표 예시(논문 Table. 1 요지)**

| Landmark | Registered Face (X,Y) | Detected A (X,Y) | Detected B (X,Y) |
|:---:|:---:|:---:|:---:|
| 👁️ Left Eye | (75,150) | (77,152) | (85,160) |
| 👁️ Right Eye | (175,150) | (177,152) | (185,160) |
| 👃 NoseBase | (125,225) | (127,227) | (135,235) |

예시에서 Left Eye 거리가 작은 A는 로그인 가능, B는 불가와 같은 논리로 설명된다(논문 본문 수치 참고).

#### 3.2.2 운동 자세 교정 🦴

**MediaPipe Pose Detection**으로 실시간 자세를 측정하고, 관절 랜드마크 좌표로 **관절 각도**를 분석한다. 특정 부위 각도가 기준과 일치하는지 판단해 **자세 교정**을 지원한다.

본 논문은 자세 교정 방법으로 다음을 비교한다.

| 방법 | 특징 | 본 연구 선택 |
|:---|:---|:---:|
| 📏 **룰베이스** | 기준 각도와 실제 각도 비교 → 실시간 오류 감지 | ✅ 채택 |
| 🧠 **BPE(임베딩)** | 시계열·유사도 중심, 완료 후 점수 제공 성향 | ❌ 실시간 교정 목적에 부적합 판단 |

또한 **자세 교정**과 **실제 운동**을 동시에 하면 사용자가 피드백 확인에 부담을 느낄 수 있어, **모드를 분리**하여 개발하였다.

**시각화(논문 Fig. 4 요지)**

| 상태 | 색 | 의미 |
|:---:|:---:|:---|
| 🟢 | 초록 | 정확한 자세(허용 범위 내) |
| 🔴 | 빨강 | 잘못된 자세 |
| ⚫ | 회색 | 인식되지 않음 |

운동별로 **우선 교정 부위**를 선정하고, 중요 부위는 **오차 범위를 좁게**, 보조 부위는 **넓게** 설정하여 체형 다양성을 수용한다. 세부 기준은 논문 **Table. 2 ~ Table. 5**에 제시된다.

**📌 스쿼트 (Table. 2 요지)**

| 부위 | 연결(개념) | 기준 각도 / 허용(예시) |
|:---|:---|:---|
| 머리 | ear–shoulder–hip | 180° (±50°) |
| 척추 | shoulder–hip–knee 등 | 75° (±20°) |
| 팔 | shoulder–elbow–wrist | 180° (±30°) |
| 다리 | hip–knee–ankle | 65° (±20°) |

**📌 플랭크 (Table. 3 요지)**

| 부위 | 연결(개념) | 기준 각도 / 허용(예시) |
|:---|:---|:---|
| 머리 | eye–shoulder–hip | 180° (±50°) |
| 척추 | eye–shoulder–hip | 180° (±20°) |
| 팔꿈치 | shoulder–elbow–wrist | 90° (±20°) |

**📌 런지 (Table. 4 요지)**

| 부위 | 연결(개념) | 기준 각도 / 허용(예시) |
|:---|:---|:---|
| 머리 | ear–shoulder–hip | 180° (±50°) |
| 척추 | ear–shoulder–hip | 180° (±20°) |
| 팔 | shoulder–elbow–wrist | 60° (±40°) |
| 다리 | hip–knee–ankle 등 | (좌/우 예시) 90°/100° (±20°) |

**📌 밸런스 (Table. 5 요지)**

| 부위 | 연결(개념) | 기준 각도 / 허용(예시) |
|:---|:---|:---|
| 머리 | ear–shoulder–hip | 180° (±50°) |
| 척추·다리 | ear–hip–knee 등 | 150° (±30°) |
| 팔·손 | elbow–thumb 등 | 180° (±20°) 등 |

교정 메시지는 “조금 더 굽혀주세요 / 덜 굽혀주세요”처럼 **방향성**을 주어 음성·영상으로 실시간 교정을 돕는다.

**난이도**는 사용자 선택에 따라 **쉬움·보통·어려움**으로 설정할 수 있으며, 운동 횟수(스쿼트·런지)와 유지 시간(플랭크·밸런스)을 달리 설정할 수 있다.

**반복 운동(스쿼트·런지) 카운팅(논문 예시)**  
- 예: 스쿼트에서 다리 각도 **160° 이상**을 서 있는 상태, **100° 이하**를 앉은 상태로 인식  
- **160° 이상 → 100° 이하 → 160° 이상**으로 복귀하면 **1회** 증가

> 📌 **Fig. 5** 실제 운동 화면(논문 원본 그림 참조)

**유지 운동(플랭크·밸런스) 정확도(식 (2))**  
설정 시간 내 기준 자세와 일치한 시간 비율로 정확도를 평가한다(논문 식 (2) 참조).  
머리·몸통·팔·다리 중 하나라도 벗어나면 시간에 비례해 정확도가 차감된다.

### 3.3 유산소 게임 애플리케이션 🎮

#### 3.3.1 아두이노 에어마우스 🖱️

유산소 게임은 **아두이노–안드로이드 블루투스** 기반 **에어마우스·움직임 감지**로 구현한다.

**하드웨어 구성**

| 구성 | 설명 |
|:---|:---|
| 🧩 아두이노 우노 | 메인 보드 |
| 📡 MPU-6050 | 3축 자이로 + 3축 가속도 |
| 🔋 배터리 | 전원 |
| 📶 HC-06 | 블루투스 모듈 |
| 🔘 버튼 | 입력 |

아두이노는 MPU-6050 데이터를 읽고 **SoftwareSerial**로 블루투스와 통신하며, 약 **2Hz** 간격으로 안드로이드에 전송한다. 안드로이드는 수신 데이터로 **마우스 포인터**를 표시한다.

가속도는 **움직임 정도**, 자이로는 **회전·방향** 추정에 사용한다. 가속도 임계값 초과 시 이벤트를 전송하는 예로, 천천히 걷기 **🚶 walk**, 빠르게 뛰기 **🏃 run** 등이 언급된다.

자이로 원시값 보정은 논문 **식 (3)**, 움직임 속도 추정은 **식 (4)** 형태로 제시된다.
